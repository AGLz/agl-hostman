# Global Conventions

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle global conventions.

## When to use this skill:

- Organizing project directory structure and file organization
- Writing or updating README files, CONTRIBUTING guides, or documentation
- Creating commit messages, branch names, or pull request descriptions
- Managing environment configuration and .env files
- Handling dependencies (package.json, requirements.txt, Gemfile)
- Setting up feature flags for incomplete features
- Maintaining changelogs or release notes
- Implementing code review processes
- Ensuring secrets and API keys are never committed
- Defining testing requirements and coverage expectations
- Working with project root files or configuration directories
