Use this skill when creating or modifying UI components, forms, buttons, navigation elements, modals, dialogs, or interactive widgets in files like React components (.jsx, .tsx), Vue templates (.vue), HTML files, or any frontend component code. Apply when implementing semantic HTML elements (nav, main, button, article, section) that convey meaning to assistive technologies, setting up keyboard navigation with visible focus indicators for all interactive elements, ensuring sufficient color contrast ratios (4.5:1 for normal text, 3:1 for large text) and not relying solely on color to convey information, writing descriptive alternative text for images and meaningful labels for all form inputs, using ARIA attributes to enhance complex components when semantic HTML isn't sufficient, creating logical heading structures (h1-h6) in proper hierarchical order, managing focus appropriately in dynamic content, modals, and single-page applications, testing with screen readers to verify accessibility, creating accessible data tables, charts, or visualizations, implementing touch-friendly controls for mobile users, or ensuring all interactive elements meet minimum size requirements (44x44px). Essential for building inclusive interfaces that work for users with disabilities, meeting legal accessibility requirements (ADA, Section 508), supporting keyboard-only navigation, ensuring compatibility with screen readers (NVDA, JAWS, VoiceOver), and providing equal access to all users regardless of abilities.

# Frontend Accessibility

This Skill provides Claude Code with specific guidance on how to adhere to coding standards as they relate to how it should handle frontend accessibility.

## When to use this skill:

- Creating or modifying UI components and interfaces
- Implementing form inputs, buttons, and interactive elements
- Building navigation menus, modals, and dialogs
- Setting up keyboard navigation and focus management
- Adding ARIA attributes for complex widgets
- Ensuring proper heading hierarchy (h1-h6)
- Writing alternative text for images and icons
- Implementing color contrast requirements
- Testing with screen readers or accessibility tools
- Creating accessible data tables, charts, or visualizations
- Working with React components, Vue templates, HTML files, or any UI code
