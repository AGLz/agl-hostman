# Global Commenting

This Skill provides Codex with specific guidance on how to adhere to coding standards as they relate to how it should handle global commenting.

## When to use this skill:

- Adding comments to explain complex logic or algorithms
- Documenting why specific implementation choices were made
- Clarifying non-obvious business rules or constraints
- Adding high-level comments for large code sections
- Avoiding over-commenting or redundant explanations
- Removing outdated comments about temporary fixes or changes
- Writing evergreen documentation that remains relevant long-term
- Preferring self-explanatory code structure and naming over comments
- Working with any code files where clarification is needed
- Reviewing existing comments for relevance and accuracy
