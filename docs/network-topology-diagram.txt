╚══════════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────────┐
│  1. NFS v4.2 - FILE-LEVEL STORAGE (Port 2049)                               │
└─────────────────────────────────────────────────────────────────────────────┘

    AGLSRV1:/mnt/storage/remote/{host}/data  ──NFS4──►  /mnt/aglsrv1/data
    AGLSRV1:/mnt/storage/shared/iso          ──NFS4──►  /mnt/aglsrv1/iso (RO)
    AGLSRV1:/mnt/storage/shared/templates    ──NFS4──►  /mnt/aglsrv1/templates (RO)

    Use Cases:
    ✓ VM disk images (alternative to iSCSI)
    ✓ Container rootfs
    ✓ ISO/template library (shared read-only)
    ✓ Migration staging area
    ✓ User/application data

    Performance: 250-400 MB/s read, 200-350 MB/s write
    Latency: 5-15ms over Tailscale VPN


┌─────────────────────────────────────────────────────────────────────────────┐
│  2. iSCSI - BLOCK-LEVEL STORAGE (Port 3260)                                 │
└─────────────────────────────────────────────────────────────────────────────┘

    Target: iqn.2025-01.local.aglsrv1:storage
    Portal: 100.x.x.x:3260
    Auth: CHAP (per-host credentials)

    LUN Mapping:
    /mnt/storage/iscsi/lun-aglsrv6-001.img   ──iSCSI──► /dev/sdb (AGLSRV6)
    /mnt/storage/iscsi/lun-aglsrv6b-001.img  ──iSCSI──► /dev/sdb (AGLSRV6b)
    /mnt/storage/iscsi/lun-fgsrv5-001.img    ──iSCSI──► /dev/sdb (FGSRV5)
    /mnt/storage/iscsi/lun-fgsrv6-001.img    ──iSCSI──► /dev/sdb (FGSRV6)

    Use Cases:
    ✓ High-performance VM disk images
    ✓ Database storage (MariaDB, PostgreSQL)
    ✓ Applications requiring block-level access
    ✓ LVM-based storage pools

    Performance: 300-500 MB/s read, 250-400 MB/s write
    Latency: 3-10ms over Tailscale VPN


┌─────────────────────────────────────────────────────────────────────────────┐
│  3. PROXMOX BACKUP SERVER - DEDUPLICATION & COMPRESSION (Port 8007)         │
└─────────────────────────────────────────────────────────────────────────────┘

    PBS Datastores:
    remote-aglsrv6   → /mnt/storage/remote/aglsrv6/backups
    remote-aglsrv6b  → /mnt/storage/remote/aglsrv6b/backups
    remote-fgsrv5    → /mnt/storage/remote/fgsrv5/backups
    remote-fgsrv6    → /mnt/storage/remote/fgsrv6/backups

    Features:
    ✓ Incremental backups with chunked deduplication
    ✓ End-to-end encryption (AES-256-GCM)
    ✓ Compression (zstd, lz4)
    ✓ Automatic garbage collection
    ✓ Backup verification jobs
    ✓ Retention policies (7 daily, 4 weekly, 6 monthly)

    Backup Schedule:
    AGLSRV6:  Daily @ 02:00 → GC @ 02:00, Prune @ 03:00, Verify @ 04:00
    AGLSRV6b: Daily @ 02:00 → GC @ 02:15, Prune @ 03:15, Verify @ 04:15
    FGSRV5:   Daily @ 02:00 → GC @ 02:30, Prune @ 03:30, Verify @ 04:30
    FGSRV6:   Daily @ 02:00 → GC @ 02:45, Prune @ 03:45, Verify @ 04:45

    Performance: 150-300 MB/s (with dedup/compression)


┌─────────────────────────────────────────────────────────────────────────────┐
│  4. SSH/SFTP - AD-HOC TRANSFERS & MIGRATIONS (Port 22)                      │
└─────────────────────────────────────────────────────────────────────────────┘

    Use Cases:
    ✓ Emergency data transfers
    ✓ Configuration synchronization
    ✓ Manual file copies (scp, rsync)
    ✓ Administrative tasks

    Performance: 50-150 MB/s (CPU-bound encryption)

║                          SECURITY ARCHITECTURE                               ║
╚══════════════════════════════════════════════════════════════════════════════╝

┌─────────────────────────────────────────────────────────────────────────────┐
│                        MULTI-LAYER SECURITY MODEL                            │
└─────────────────────────────────────────────────────────────────────────────┘

Layer 5: Application Security
├── PBS: Token-based authentication (API tokens)
├── SSH: Public key authentication (no passwords)
└── NFS/iSCSI: ACL-based access control

Layer 4: Protocol Security
├── iSCSI: CHAP authentication (per-host credentials)
├── NFS: sec=sys (Tailscale provides encryption)
└── PBS: TLS 1.3 + Certificate pinning

Layer 3: Transport Security
├── Optional IPSec for iSCSI (if needed)
└── TLS 1.3 for PBS API

Layer 2: Network Security
├── Tailscale WireGuard: ChaCha20-Poly1305 encryption
├── Zero-Trust network model
└── ACL-based access control per service/port

Layer 1: Host Security
├── UFW firewall: Deny-by-default, allow Tailscale subnet
├── SELinux/AppArmor (optional)
├── Automatic security updates
└── Audit logging (auditd)
