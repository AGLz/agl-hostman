PROMOTION WORKFLOW EXAMPLE
================================================================================

1. CREATE PROMOTION REQUEST (from QA):
   curl -X POST https://qa-agl.aglz.io/api/promotion/qa-to-uat \
     -H "Authorization: Bearer TOKEN" \
     -d '{"source_version": "qa-1a2b3c4", "notes": "Sprint 15"}'
   
   Response: {"promotion_id": "uuid", "status": "pending"}

2. APPROVE PROMOTION (admin):
   curl -X POST https://qa-agl.aglz.io/api/promotion/{id}/approve \
     -H "Authorization: Bearer ADMIN_TOKEN" \
     -d '{"approval_notes": "Approved", "auto_deploy": true}'
   
   Response: {"status": "approved", "deployment_id": "xxx"}

3. GITHUB ACTIONS DEPLOYS:
   - Builds Docker image: uat-{sha}
   - Pushes to Harbor: agl-hostman-uat
   - Deploys to CT181 via Dokploy
   - Runs smoke tests (12 tests)
   - Updates promotion status

4. VERIFY COMPLETION:
   curl https://qa-agl.aglz.io/api/promotion/{id}/status
   
   Response: {
     "status": "completed",
     "smoke_test_results": {"total": 12, "passed": 12, "failed": 0}
   }
