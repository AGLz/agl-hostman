Authentication

The AGL API uses multiple authentication methods to support different use cases.

JWT tokens are the recommended authentication method for user accounts.

curl 
curl -X GET https://api.agl.com/api/infrastructure/status \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json"
JavaScript 
const response = await fetch('https://api.agl.com/api/infrastructure/status', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});

Quick Start

POST /auth/login

Authenticate and receive a JWT token.

{
  "email": "admin@agl.com",
  "password": "your-password",
  "remember": true
}

Response:

{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer",
  "expires_in": 3600,
  "user": {
    "id": 1,
    "name": "Admin User",
    "email": "admin@agl.com",
    "roles": ["admin"]
  }
}

Common Endpoints

GET /infrastructure/status Auth

Get current infrastructure status including servers, containers, and VMs.

GET /monitoring/metrics?period=24h Auth

Retrieve monitoring metrics for specified time period.

POST /ai-service/predict Auth

Make AI-powered predictions using available models.

GET /alerts/active Auth

Get all active alerts with severity levels.
